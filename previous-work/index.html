VLM related / Project 01

Large-Scale Image and Video Annotation Pipeline

Designed a high-throughput annotation and filtering pipeline to curate image and video data for a VLM deployed on smart glasses, enabling concise, high-fidelity answers to visual queries.

Scale Thousands of V100/A100 GPUs
Media First-party and third-party media
Throughput Billion samples per day

The work focused on turning raw, heterogeneous web and internal media into high-value training examples while respecting privacy and quality constraints.

Technical emphasis

The system coordinated concurrent GPU inference with high throughput data loading and writing, then composed multiple model passes into a repeatable enrichment graph.

Pipeline stages
  1. Start from first-party and third-party raw media.
  2. Filter by visual quality: resolution, brightness, blurriness, temporal difference, aesthetics, and related signals.
  3. Apply privacy mitigation through face blurring.
  4. Run named entity recognition on original captions.
  5. Run classification and object detection on original images.
  6. Re-caption images and videos.
  7. Score CLIP similarity.
  8. Evaluate quality and run ablations.

VLM related / Project 02

70B Vision-Language Model Pretraining

Contributed to a LLaVA-style 70B VLM pretraining program that first built a strong vision encoder with OpenCLIP-style contrastive training, then connected it to a large language model through aligner training over the frozen encoder and backbone.

Model class 70B VLM
Training style Contrastive encoder + aligner
Dataset Curated image-caption pairs

The core idea was to improve visual grounding before alignment: build a strong visual representation, then teach the language model how to use it.

Technical emphasis

Stage one trained the vision encoder with OpenCLIP-style contrastive learning on curated image-caption pairs, scaling image resolution progressively and ablating encoder architectures. Stage two froze the encoder and the language backbone, training only the aligner to map visual embeddings into the language token space.

Training stages
  1. Train the vision encoder with OpenCLIP-style image-text contrastive learning on curated pairs.
  2. Scale image resolution progressively and ablate encoder architectures to pick the strongest backbone.
  3. Freeze the trained vision encoder and the 70B language model.
  4. Train the aligner to project visual embeddings into the language token space, LLaVA style.

Llama 4 full-duplex speech / Project 03

Speech Steerability and Persona Control

Developed post-training data patterns for a full-duplex speech model so users could control response style through voice and steer the assistant across more than 20 personas.

Capability Voice-driven style control
Coverage 20+ personas
Data mode Human voice + TTS + voice cloning

The project made persona changes feel like a natural part of a spoken conversation instead of a brittle prompt-only instruction.

Technical emphasis

Data design covered normal conversations, persona rewrites, persona transition instructions, and multi-speaker sessions where user and assistant could carry different styles.

Post-training recipe
  1. Start with neutral conversational examples.
  2. Rewrite assistant responses into target personas.
  3. Create explicit transitions between personas, including return-to-normal cases.
  4. Record voice actors, synthesize variants, and preserve speaker/persona separation.

Llama 4 full-duplex speech / Project 04

Realtime Barge-In Behavior

Designed training data that allowed users to interrupt the assistant mid-response, producing more natural full-duplex conversational behavior during realtime speech interaction.

Capability Natural interruption
Timing 0.1s to 1s cutoff windows
Evaluation Ablation-driven breakpoint selection

For example, the assistant is mid-sentence — “Good morning, today is a sunny day, temperature is 25 degrees celsius…” — when the user cuts in at “25” to ask “What’s the weather in San Francisco?”. The assistant stops shortly after, dropping the rest of its answer instead of talking over the user.

Technical emphasis

Starting from a normal turn, the data construction selected a breakpoint inside the assistant audio (here, at the word “25”), overlaid the user’s next utterance, then discarded the trailing assistant audio after a short 0.1–1s cutoff so the model learns to taper off naturally.

Data construction
  1. Start from a normal turn: the assistant answers “…temperature is 25 degrees celsius, you don’t need to bring an umbrella.”
  2. Select a breakpoint inside the assistant turn — the user barges in at “25”.
  3. Overlay the user’s utterance (“What’s the weather in San Francisco?”) on top of the assistant audio.
  4. Cut and dim the remaining assistant audio 0.1–1s after the breakpoint, discarding the “umbrella” tail, and train on the interrupted session.

Avocado speech model / Project 05

Multilingual ASR Pretraining Data Curation

Built a very large-scale curation pipeline for an in-house ASR model targeting written-form transcription across more than 40 priority languages and 200M+ hours of speech.

Scale 200M+ hours
Parallelism Peak 10k GPUs
Languages 40+ priority languages

The work emphasized practical multilingual coverage, especially the messy cases where language boundaries blur inside one speech segment.

Technical emphasis

The pipeline combined VAD, LID, model-routed ASR, multilingual alignment, and diarization with infrastructure improvements for vLLM serving, length-based packing, and Blackwell GPU support.

Curation stages
  1. Detect speech regions with VAD.
  2. Run language identification, including code-switching ablations.
  3. Route ASR workloads to language-appropriate models.
  4. Pack audio by length to reduce padding waste and improve GPU utilization.
  5. Apply multilingual alignment and speaker diarization.

Avocado speech model / Project 06

Multilingual Dubbing Post-Training and Data Curation

Curated and post-trained an offline speech-and-text to speech translation model supporting more than 20 priority languages, with explicit control over transcript, translation, timing, and speaker similarity.

Task Speech translation and dubbing
Coverage 20+ target languages
Optimization GRPO reward design

The model learned to preserve meaning and speaker identity while producing translated speech tokens, with surprisingly strong generalization to singing.

Technical emphasis

Curation began from IG Reels filtered to clips that are more than 50% speech, then ran LID and ASR before fanning out into 20+ target-language translations — each scored against the source transcription with a MetricX model and enriched with alignment and diarization. Post-training examples interleaved the translated text with discrete speech tokens that reconstruct into waveform, optimized with GRPO rewards for format, duration, transcription consistency, translation quality, speaker similarity, and accent control. 

user: [audio tokens] This is the transcription of the
      English speech: "Good morning, how are you today?"
      Translate the English speech into Chinese.

assistant:
  Transcription: Good morning, how are you today?
  Translation:   早上好，你今天过得怎么样？
  Interleaved text + speech tokens:
      早上好 <vq0><vq1> ，你今天 <vq2><vq3> 过得
      怎么样 <vq4><vq5> ？

# <vqN> are discrete speech tokens decoded back into a
# translated, speaker-matched Chinese waveform.
Training and reward design
  1. Start from IG Reels and keep clips that are more than 50% speech.
  2. Run LID, then ASR to transcribe the source (e.g. English “Good morning, how are you today?”).
  3. Fan out into 20+ target-language translations (Chinese, Hindi, Japanese, Spanish, and more).
  4. Score each translation against the source transcription with MetricX, then append alignment, diarization, and quality tags.
  5. Post-train with interleaved text and discrete speech tokens, using GRPO to reward format, duration, content match, speaker similarity, and native-sounding output.
