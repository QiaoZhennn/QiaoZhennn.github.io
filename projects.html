Introduction to the PHM Toolbox

PHM Toolbox is my graduation project. It was developed with Matlab when I did my internship in Hangzhou AIMS company.

Introduction to the online shopping website

This a small online shopping website but with nearly whole basic function compared with common online shopping website.

First part is the background management system. The administrator is able to add or drop merchandises. Besides, they can add or drop the attributes of a merchandise. And, they can manually copy data from database to the Redis cache, so that customers can look through merchandises more efficiently.

Second part is the customer trading system. A customer can look through all the merchandises, or with some filters, or rank them by some order. In addition, They are able to search the database or cache with some keywords, Apache Solr is used to implement this function. First, they need to add merchandise to the shopping cart. Second, they go to the shopping cart and make sure what they want to buy and where they want to deliver, forming an order. Third, pay the bill. This website can further use a public online finance interface to accomplish the payment process.

Introduction to the crowd funding website

This is a online crowd funding website. People can publish ideas and collect money on this website.

First part is the background management system. Web administrator is able to control permissions of different types of user.

Second part is the user authentication workflow. Activiti5 is used to design and control the workflow. A user need to choose its user type firstly. And then, start the procedure of authentication. There are 4 main steps of user authentication. First, basic information of user. Second, upload certification files, which are defined by the background management system. Third, email confirmation. Fourth, user will receive an email. Filling the verification code to confirm this user authentication application. After finishing all these steps, an web administrator can check all the information provided by this user and decide whether pass this application or not.

Introduction to the online survey website

This is an full function online survey website but without a colorful portal. I was Mainly focus on backend development.

First part is background management system. Web administrators have various permissions. They can view the result of each survey, manage access permission of different user roles, view logs of the website.

There are two types of user: enterprise user and individual user. Enterprise users can design surveys and participate in surveys, while individual users can only participate in surveys.
