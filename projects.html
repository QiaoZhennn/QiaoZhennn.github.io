Introduction to the online shopping website

This a small online shopping website but with nearly whole basic function compared with common online shopping website.

Second part is the customer trading system. A customer can look through all the merchandises, or with some filters, or rank them by some order. In addition, They are able to search the database or cache with some keywords, Apache Solr is used to implement this function. First, they need to add merchandise to the shopping cart. Second, they go to the shopping cart and make sure what they want to buy and where they want to deliver, forming an order. Third, pay the bill. This website can further use a public online finance interface to accomplish the payment process.

Main techniques: 1. Built the background seller management system and foreground customer trading system using Spring + SpringMVC + Mybatis (SSM) framework with Java . 2. Implemented a web service for user authentication using Apache CXF + Spring framework with Java. 3. Designed background seller management user interface using EasyUI with JavaScript and CSS. 4. Used Redis as cache to improve efficiency. 5. Implemented keyword search function using Apache Solr with Java.

First part is the background management system. The administrator is able to add or drop merchandises. Besides, they can add or drop the attributes of a merchandise. And, they can manually copy data from database to the Redis cache, so that customers can look through merchandises more efficiently.

Introduction to the crowd funding website

This is a online crowd funding website. User can publish ideas and collect money to start their ideas through this website.

Main techniques: 1. Built background management system including user register system, role-based access control (RBAC) and user real name authentication using SSM framework with Java. 2. Implemented user authentication workflow function using Activiti5 with Java. 3. Designed user interface using Bootstrap3.

First part is the background management system. Web administrator is able to control permissions of different types of user.

Demonstration of background management system

Introduction to the crowd funding website

Second part is the user authentication workflow. Activiti5 is used to design and control the workflow. A user needs to choose its user type firstly. And then, start the procedure of authentication. There are 4 main steps for user authentication. First, fill user basic information form. Second, upload certification files, which are defined by the background management system. Third, confirm email address. Fourth, user will receive an email. Filling the verification code to confirm this user authentication application. After finishing all these steps, an web administrator can check all the information provided by this user and decide whether pass this application or not.
